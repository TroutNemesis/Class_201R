  • Th - Sep 10: Lab Section #1 Resume:

    Basic HTML programming expertise is an important skill for Computer Science graduates.

    In creating a web page you can think of the Content Layer as being the basic HTML. The Presentation Layer is the CSS, and the Behavior Layer as the javascript. We wont be dealing with the Behavior layer until late in the class, but this assignment will help you become familiar with the Content Layer and the Presentation Layer.

    You should complete the codecademy resume assignment. Complete as much of the codecademy HTML & CSS exercises as you need, but at least go through the material you will need to reference an external CSS file.

    You should then upload your html and CSS file to your koding.com web site. Submit the URL on your koding.com web site that will access your resume.

    Your page should have at least the following:

    • lists
    • different background colors for different sections
    • images
    • links to other pages
    • headings that are defined in your CSS

    Passoff

    • Submit the URL for your koding.com web page in the learningsuite submission along with a github.com or bitbucket repository with all of the files needed to make your page work.

  • T - Sep 22: Lab Section #2 Jquery and AJAX:

    First take the Khan quiz to make sure you know javascript.

    Then complete the codecademy section on events to create a news reader.

    Get the news reader working on your koding.com site.

    Now, you will implement a jquery ajax call to get the current weather. Add this to the bottom of your news reader.

    The weather section should do the following:

    1. Allow the user to type characters into a form. With each keystroke, query a REST service that will return a list of cities in utah that have those characters as a prefix. This will provide a hint to the user as to what legal values can be. Put these options into a list. These hints may be helpful.
    2. When the user selects the submit button, take the entry from the form and put it into the CITY box on the page. These hints may be helpful.
    3. The submit button will also query the weatherunderground REST service to get the current weather for that city. These hints may be helpful.

    Submit the URL where your code can be found along with any comments you have that will make your assignment easier to grade.

    Grading:

    • (40%) You have the newsreader from codecademy working
    • (10%) As you type characters in the weather section, the application suggests legal values.
    • (10%) When the submit button is pressed, you correctly fill in the City box.
    • (40%) You make the request to the weatherunderground site and fill in the weather

  • T - Oct 13: Lab Section #4 Node.js:

    In this lab you will implement the cities RESTful service in Node.js that connects to your weather AJAX application. You will take a URL with a ?q=chars suffix and will look up all of the utah cities that have chose chars as a prefix. You will then return a JSON string corresponding to the entries. The first step in this lab is to get the web server running from section 7.1 in the book (you may find the node documentation helpful). You should be able to download all of the static pages for your application using this web server. Test your server using the static pages we used to test your C++ web server. Then you should copy your jquery javascipt weather application into the html folder of the node web server. You should be running this web server on the Amazon CS2 server or another virtual server. The jquery application should be modified to make the jsonp request to your own REST server instead of the one used for your jquery lab. These hints may be helpful if you get stuck. You may want to upload your server to a git repository and create a npm repository for your code, but this is not required. Passoff: You should test your server to make sure it works with your jquery script. Your submission to learningsuite should contain:

    • The URL of your jquery script
    • The URL of your REST server

    The TAs will need to access the URL of your server, so you need to have it run even after you log out. Try using nohup to make this happen:

    nohup sudo node static.js >& output.node &

    In this command you are disassociating the process from your login, so it will not exit when you logout. The >& means redirection stdout and stderr to the file output.node. The & at the end means to run it in background. Grading:

    • (20%) Your node.js web server can return the test files from the prior labs (test1.html, test2.txt, test3.gif, test4.jpg).
    • (30%) Your node.js service correctly returns JSON entries for cities.
    • (50%) You return the correct cities with prefix given in the ?q=chars string.

  • Th - Oct 22: Lab Section #5 MongoDB:

    Many social media applications allow users to insert comments. In this lab, you will configure a MongoDB database and connect it to your node.js web server.

    Modify your jquery application to display entries from the database and allow users to enter new comments. Your output should look something like this:

    ===========================

    • {"Name":"Foobar","Comment":"Hello"}
    success

    Current Comments:

    • Name: Mickey Comment: Hello
    • Name: Mark Comment: This is a comment
    • Name: Frodo Comment: Hi
    • Name: Minnie Comment: Hello
    • Name: Foobar Comment: Hello

    ===========================

    When you click the "submit button", fill in a div with the JSON string you will send to the POST interface of your REST service. When the REST service returns, fill in the next div with "success". When the user clicks "show comments", contact the GET interface of your REST service and retrieve all of the comments currently in the database and print them in an unordered list.

    These hints may be helpful as your implement your node/mongodb application.

    Passoff:

    You should test your server to make sure it works with your jquery script. Your submission to learningsuite should contain:

    • The URL of your jquery script
    • The URL of your REST server for comments>
    • The URL of the github repository with all of your source

    The TAs will need to access the URL of your server, so you need to have it run even after you log out. Try using nohup to make this happen:

    nohup sudo mongod --fork --logpath /var/log/mongodb.log >& mongo.out

    nohup sudo node static.js >& output.node &

    In this command you are disassociating the process from your login, so it will not exit when you logout. The >& means redirection stdout and stderr to the file output.node. The & at the end means to run it in background.

    Grading:

    • (30%) The GET interface for your REST service returns the current comments
    • (40%) You can pass the POST interface for your REST service a new comment and it correctly inserts it.
    • (30%) Your jquery application displays things properly.
